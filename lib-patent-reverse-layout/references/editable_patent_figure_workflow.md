# 可编辑专利附图生成与批注迭代流程

本参考用于锂离子电池专利的反向附图生成，尤其适用于“参照 A 专利附图格式，理解 B 专利保护点，输出可编辑附图”的任务。

## 1. 触发场景

出现以下需求时调用：

- 参照某专利附图格式生成另一专利附图。
- 根据权利要求、说明书或实施例重新理解保护点后绘图。
- 将已有 PNG/SVG 附图改成 PPT 可编辑元素。
- 按 PPT 批注修订附图。
- 用户明确要求“PPT 保留各组成元素”“可框选修改”“不是整张图片”。

## 2. 输入解析

优先读取 `.docx` 的以下内容：

1. 权利要求书：抽取独权核心结构、尺寸关系、材料和位置关系。
2. 附图说明：抽取图号、各图主题和标号体系。
3. 具体实施方式：抽取“为便于理解，如图……”中的结构解释。
4. 实施例：抽取典型值、范围端点和对比例边界。
5. `word/media/`：导出原附图，用于格式对齐和错误对照。

对每个尺寸符号建立定义表：

| 字段 | 内容 |
|---|---|
| 符号 | 如 L1、L2、W1、W2、T1、t1 |
| 原文定义 | 原文完整语义 |
| 方向 | 平行/垂直于极片端边、厚度方向、长度方向等 |
| 状态 | 安装状态、剖面状态、展开状态、测量状态 |
| 保护作用 | 比值关系、阈值、材料/结构耦合 |
| 绘图方式 | 尺寸箭头、展开小图、剖面厚度、图例说明 |

## 3. 参考格式迁移

从参考专利提取“画法”，不得机械继承“保护点”：

- 图号和标题格式。
- 黑白线稿风格。
- 标号数字 + 括号结构名称。
- 引线和尺寸箭头。
- 图例填充：黑色、灰色、斜线、点状、虚线。
- 剖面中层次顺序和图例说明。

迁移时必须重写目标案的尺寸含义。例如参考案的 `L1/L2` 可能是折返长度，目标案的 `W1/W2` 可能是展开宽度，不能沿用同一画法。

## 4. 保护点到图形的映射

绘图前输出或内部维护映射表：

| 保护点 | 附图元素 | 标号 | 标注方式 | 依据 |
|---|---|---|---|---|
| 正极片 | 长条极片 | 1 | 中央文本 | 权利要求/实施方式 |
| 负极片 | 长条极片 | 2 | 中央文本 | 权利要求/实施方式 |
| 隔膜 | 虚线长条 | 3 | 中央文本 | 权利要求/实施方式 |
| 第一端部绝缘件 | 跨端边包覆结构 | 41 | 黑色填充 | 权利要求/实施方式 |
| 第二端部绝缘件 | 跨端边包覆结构 | 42 | 灰色填充 | 权利要求/实施方式 |
| 第三/第四绝缘件 | 单面或微折返结构 | 43/44 | 斜线填充 | 从权/实施方式 |
| 圆角凹口 | 侧边圆弧凹口 | 51 | 弧线断口 | 从权/实施方式 |

## 5. 展开宽度与折返长度的区分

必须严格区分：

- **折返长度**：安装状态下，绝缘件从极片端边向单侧表面覆盖的长度。
- **展开宽度**：绝缘件沿垂直于极片端边方向展开后的总宽度，可包含两表面覆盖段及端边跨越段。

若原文写“展开宽度 W1/W2”，不得只在剖面单侧覆盖段上标 `W1/W2`。推荐画法：

1. 主结构图中保留包覆安装状态，用标号指向端部绝缘件。
2. 在图中另设“展开状态小图”，用一条独立条带和尺寸箭头表示 `W1`、`W2`。
3. 局部剖视图中，安装状态的水平覆盖段标注为“单侧覆盖段”或不标尺寸。
4. 局部剖视图上方或旁侧另画展开状态，标注“W1（展开宽度，范围）”。

示例判断：

- `L1/L2=1.3~3.5` 且原文定义为折返长度：可标在单侧覆盖段。
- `W1/W2=0.50~1.10` 且原文定义为展开宽度：必须标展开总宽度。

## 6. 局部剖视图层次规则

剖面图应避免装饰性复杂化，优先表达保护点：

- 正极片可用“活性物质层/集流体/活性物质层”三层。
- 端部绝缘件可用“基材/胶层”两层。
- 胶层应贴近被粘附表面或极片端边，基材位于外侧；如原文另有定义，从原文。
- 厚度标注 `t1/T1` 应指向被限定对象的厚度，不得误画成整个剖面高度。
- 若厚度范围仅限定端部绝缘件本身，标注文本应写“第一端部绝缘件厚度……”而非泛称“总厚度”。

## 7. PPT 原生形状输出规则

当用户要求 PPT 可编辑时：

1. 不得将整张 SVG/PNG 作为最终 PPT 图页插入。
2. 使用 PPT 原生元素：矩形、线条、弧线/折线、文本框、连接线、尺寸箭头、图例。
3. 每个标号、引线、图例块和尺寸箭头均应可单独框选修改。
4. 活性物质层点状填充不要过密，以免对象过多影响编辑。
5. 圆角凹口优先用“白色断口 + 多段弧线/自由曲线”绘制，不用椭圆遮罩造成凸凹方向误读。
6. 若需要预览，可另输出 PNG；PNG 只能作为预览，不作为最终可编辑 PPT 的主体。

必须执行校验：

```text
ppt/media 文件数 = 0
每页 picture shape 数 = 0
```

若校验不通过，说明该 PPT 不是合格的“原生形状可编辑附图”。

## 8. 批注迭代流程

收到 PPT 批注后：

1. 读取批注目标的 slide、shape id、文本、坐标、尺寸。
2. 判断问题类型：重叠、标注错误、结构错误、层位错误、删除标注、尺寸语义错误。
3. 修改生成脚本或矢量母版，重新生成文件；避免只在 PPT 中手动拖动。
4. 若原文件被打开占用，生成 `_修订版` 新文件，不强行覆盖。
5. 修订后复查：
   - 无重叠；
   - 无引线穿字；
   - 无尺寸箭头压标号；
   - 无图例遮挡；
   - 无错层或错标；
   - PPT 原生性校验通过。

最终回复按批注逐条说明处理结果。

## 9. 文件交付建议

推荐输出：

- `*_原生形状.pptx`：最终可编辑附图。
- `*.svg`：矢量母版，可选。
- `*.png`：预览图，可选。
- `附图说明_保护点摘要.md`：说明尺寸语义、保护点映射和修订依据。
- `.zip`：汇总交付。

若用户正在批注原 PPT，保留原文件并生成 `_修订版.pptx`，避免覆盖导致批注上下文丢失。
