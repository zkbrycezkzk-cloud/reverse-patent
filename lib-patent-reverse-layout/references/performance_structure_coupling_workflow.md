# 性能-结构联立约束反向专利工作流

本参考用于把我方/竞品实测性能数据转化为“产品对象 + 可测性能参数 + 可反向材料/结构参数 + 关系式”型权利要求，并按 Step 1 -> Step 2 -> Step 3 -> Step 4 -> Compile 分段输出。结构梯度数据充分时，可采用“性能联立约束 + 结构联立条件”的增强形式。涉及液态、凝胶、固态、准固态体系时，同时参考 `../lib-patent-refiner/references/battery_agent_keypoints.md`。

## 适用场景

优先使用本工作流的输入特征：

1. 有高倍率性能对比数据，例如过电位、平台压差、大面温度、校准温度；
2. 有可从成品拆解获得的极片、材料或隔膜参数，例如长度、厚度、面密度、压实密度、材料比例、隔膜涂层；
3. 用户希望保护我方产品，同时让竞品参数落入保护范围；
4. 用户明确要求构建协调关系式，避免权利要求1简单罗列参数。
5. 用户输入的是材料参数、极片参数和性能参数的组合数据，需要把实测值通过数据居中区间设计法嵌入权要范围中心。
6. 用户输入凝胶、固态或准固态体系的电导率、模量比、相变峰、元素梯度、质量分数或溶剂化结构数据。

## 核心权利要求模型

权利要求1优先写成以下耦合式格式：

```text
一种[锂离子电池/圆柱锂离子电池/电芯/极片]，包括[必要结构部件]，其特征在于，
所述[产品]具有可测性能参数 P1，且 P1 满足[P1范围]；
所述[产品/材料/极片结构]具有可反向检测参数 M1 和/或 S1；
其中，基于 M1、S1 和 P1 得到耦合参数 K，且 K 满足：
[下限]≤F(M1,S1,P1)≤[上限]。
```

必须优先输出至少三类候选权利要求1：

1. **性能-材料耦合式**：例如容量保持率 `x` 与元素侵入层厚度占比 `m`、侵入颗粒占比 `n` 关联。
2. **性能-电学耦合式**：例如交流内阻 `a`、直流内阻 `b`、温升系数 `c` 关联。
3. **性能-极片结构耦合式**：例如倍率温升、面密度、厚度、孔隙率、导热系数或极耳结构参数关联。

参考句式：

```text
一种锂离子电池，包括正极片、隔膜、负极片以及用于传输锂离子的电解液，
其特征在于，在测得所述锂离子电池的第一容量保持率为 x 时，78%≤x≤89%，
所述正极材料颗粒的直径为 D，且所述正极材料颗粒的 F 元素侵入层的深度为 L，
得到在所述正极材料颗粒的径向上所述 F 元素侵入层的厚度占比 m=L/D，
且 m 与 x 满足：m≤1-x；
以及统计所述负极材料颗粒中被 F 元素侵入的负极材料颗粒的占比 n，且满足：n≤(1-x)/2。
```

```text
一种圆柱锂离子电池，包括壳体、组装于所述壳体内的卷芯以及填充于壳体内的电解液，
其特征在于，所述圆柱锂离子电池的交流内阻为 a 毫欧，直流内阻为 b 毫欧，
所述圆柱锂离子电池在 10C 的放电倍率下的温升系数为 c K/(min*Ah)，
所述温升系数与所述交流内阻和直流内阻满足以下关系式：0.1≤[2a/(a+b)]*c≤4。
```

上述示例只作为格式参照。实际案件必须使用用户输入或可补测的实测参数，不得伪造元素侵入、内阻、温升或结构参数。

当已具备厚度方向孔隙率、导热系数等结构梯度数据时，可将材料/极片参数联立条件具体化为：

```text
性能联立约束：ΔV≤A，且 Ts≤B；
结构联立条件：ε(Z1)-ε(Z2)≥C，且 λ(Z2)-λ(Z1)≥D。
```

阈值选择规则：

- 独权阈值必须覆盖我方产品和拟覆盖竞品，不得因优选性能排除我方实测样品；
- 优选阈值放入从权。例如我方 ΔV=0.218 V 时，不应把 `ΔV≤0.215 V` 放入独权；可将独权设为 `ΔV≤0.222 V`，从权设为 `ΔV≤0.215 V`；
- 温升口径必须统一。若用户确认使用 `Ts≤66.5℃`，全稿不得混用 `ΔT≤7.5℃` 作为权1判定口径。
- 协调关系式必须由已输入或可补测的材料/极片/性能参数构成；实测值应位于宽范围中心或优选范围中心，不能把实测值压在端点。
- 若缺少某个结构参数，不得将其写入权利要求1的必备条件；应写入待补测清单，或在用户确认后作为预言性实施例变量并标注 `[P]`。

## Z1/Z2位置定义

在权利要求和说明书中必须前置定义：

- `Z1`：活性物质层厚度方向上，自靠近隔膜的表面起向内延伸、占活性物质层总厚度 `20%-35%` 的区域；
- `Z2`：活性物质层厚度方向上，自靠近集流体的界面起向内延伸、占活性物质层总厚度 `20%-35%` 的区域；
- 双面涂布极片中，每一侧涂层分别以其自身靠近隔膜的表面和靠近集流体的界面确定 Z1/Z2；
- 每片极片至少取头部、中部、尾部三个位置，取平均值作为判定值。

## Step 1 输出

只输出权利要求设计，不进入实施例。默认输出 15 条权利要求，权利要求1为核心，并遵守 `../lib-patent-refiner/references/drafting_quality_requirements.md` 的权利要求结构。必须包含：

1. 权利要求1的耦合式草案，必须包含产品对象、可测性能参数、可反向材料/结构参数、协调关系式、排除性限定和可反向参数范围；
2. 从属权利要求树；
3. 数据居中/覆盖表；
4. 参数反向可测性评分；
5. 权要-实测值覆盖关系；
6. 待补测数据缺口。

若 `ε(Z1)-ε(Z2)`、`λ(Z2)-λ(Z1)` 尚无实测值，标为“待补测”；可在用户确认后用于预言性实施例，但正式提交前建议补测。

固定结束句：

```text
Step 1 完成。请确认是否进入 Step 2。
```

## Step 2 输出

只输出实施例和对比例。

实施例设计必须按 `../lib-patent-refiner/references/drafting_quality_requirements.md` 的单因素矩阵执行：

- 实施例1：我方实测产品；
- 实施例1必须写明反应方程式示意、称量配比、配制、装配、固化或原位固化、可反向验证点；
- 每个核心变量至少设置 2 个实施例和 1 个对比例；
- 每例必须使用“本实施例/本对比例与实施例1的区别在于……”固定差异句式；
- 实施例2：竞品参数覆盖样，正式文本中去除竞品名称；
- 实施例3-N：围绕权要变量设置单变量边界和优选结构；
- 预言性数据必须标注 `[P]`。

对比例至少覆盖：

1. 反向规避：梯度方向反转；
2. 超限规避：ε梯度低于下限；
3. 超限规避：λ梯度低于下限；
4. 替换规避：材料或颗粒比例超限；
5. 缺省规避：关键涂层或结构缺省；
6. 组合规避：多个参数组合恶化；
7. 性能阈值不达标：结构相近但 ΔV 或 Ts 超限。

固定结束句：

```text
Step 2 完成。请确认是否进入 Step 3。
```

## Step 3 输出

输出详细性能测试方法和性能数据汇总表。必须包含：

1. 40A倍率过电位测试；
2. 大面温度 `Ts` 测试，统一使用确认后的温升口径；
3. Z1/Z2孔隙率测试；
4. Z1/Z2有效导热系数测试；
5. 涵盖全部实施例和对比例的性能数据汇总表；
6. 性能-权要支撑关系；
7. 逐变量机理分析提纲。

每个变量的机理解释不少于 100 个中文字符，除化学式、实验数据、公式和单位外不得使用箭头类符号。

每个变量默认给出 3 个产品核心关联性能指标、检测方法和计算公式。凝胶或固态体系可优先选用 25℃离子电导率、循环容量保持率、倍率容量保持率、界面阻抗、DSC相变峰、DMA模量比、热收缩率或安全测试结果。

竞品 ε/λ 可在用户确认后使用参照已有数据的预言性数据，并标注 `[P]`。

固定结束句：

```text
Step 3 完成。请确认是否进入 Step 4。
```

## Step 4 输出

输出可反向性能测试/产品鉴定方法。必须包含：

1. 从一只未知成品电池出发的法证级样品登记、预处理、安全放电、方向标记和安全记录；
2. 无损成像、拆解、取样和样品分类；
3. 材料表征、结构测量、性能复测；
4. 权要-检测路径对应表；
5. 核心特征组合判定标准、定量计算公式、标准差或多点平均规则；
6. Z1/Z2多位置取样和平均值判定规则。
7. 去盐、去溶剂、空白扣除、n≥3复测、误差控制和误判预防。

固定结束句：

```text
Step 4 完成。请确认是否进入 Compile 阶段合成最终申请稿。
```

## Compile 补强

Compile 阶段应合成完整申请稿，并按 `lib-patent-refiner-v2.2` 的补强规则填充：

1. 实施例1详细制备方式；
2. 实施例2-N逐段调控方式；
3. 对比例1-M逐段调控方式；
4. 性能数据处理；
5. 逐变量机理展开；
6. 产品反向鉴定细节；
7. 覆盖性审核表；
8. 摘要。

保持已确认的权利要求和核心方案不变，只补齐说明书缺失内容。

Compile 输出前必须按 `../lib-patent-refiner/references/drafting_quality_requirements.md` 做一致性自检，尤其检查权利要求1、实施例1工艺、单因素矩阵、反向检测 SOP 和核心变量闭环。
